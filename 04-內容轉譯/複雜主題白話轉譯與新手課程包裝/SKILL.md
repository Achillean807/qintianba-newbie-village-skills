# 複雜主題白話轉譯與新手課程包裝

很多東西賣不動，不是因為內容不夠好。
是因為講的人太熟，聽的人太陌生。

## 什麼時候用

- AI / 技術主題太硬
- 顧問服務說了半天還是聽不懂
- 課程看起來很厲害，但新手不敢進來
- 服務流程太抽象，客戶無法想像自己會得到什麼
- 想把複雜知識整理成說明頁、貼文、活動介紹、課程介紹

## 核心任務

把專業語言翻譯成：
- 聽得懂
- 看得見畫面
- 知道自己適不適合
- 願意做下一步

## 白話轉譯三步驟

### 第一步：找出使用者原本在說的專業詞
例如：
- 自動化
- 工作流
- 分層漏斗
- 知識庫
- 提示工程

### 第二步：翻成生活語言
不是解釋名詞而已。
是翻成一般人能感受到的情境。

例如：
- 自動化 → 把重複事情交給系統做，省回你的人力
- 工作流 → 從有人來問，到最後成交，中間每一步怎麼接
- 知識庫 → 把散掉的資料整理成之後找得到、用得動的版本

### 第三步：補上結果畫面
只講概念很乾。
要補一句：做完之後你會少什麼麻煩、快多少、穩多少。

## 新手包裝原則

### 1. 不要把新手當白痴，也不要把新手當同行
語氣要尊重，但說法要降階。

### 2. 把路徑切短
不要一開始就端出完整世界觀。
先給第一步能做的版本。

### 3. 先講「不需要」什麼
新手最怕的是門檻。
先告訴他：
- 不需要先懂全部
- 不需要先會寫程式
- 不需要一開始就買滿裝備

### 4. 每一段都要回答一個隱性問題
- 這是什麼
- 為什麼我要管這個
- 這跟我有什麼關係
- 做了之後差在哪
- 我現在可以先做什麼

## 包裝骨架

```
1. 你現在可能遇到的場景
2. 問題不是出在你不夠努力，而是方法或理解有落差
3. 這件事其實可以用一句白話講清楚
4. 你會得到的改變是什麼
5. 第一個最小可做步驟
```

## 適合輸出的內容

### 格式 A：課程介紹
- 誰適合
- 誰不適合
- 你會學到什麼
- 你不需要先會什麼
- 下一步 CTA

### 格式 B：服務白話版介紹
- 一句定位
- 三個你會得到的結果
- 合作流程簡述
- 常見誤解拆解

### 格式 C：技術主題轉譯貼文
- Hook
- 白話說明
- 小案例
- CTA

### 格式 D：活動頁的「這堂是什麼」區塊
- 一句話版本
- 三點帶走重點
- 適合對象
- 不適合對象

## 自我檢查

- 沒背景的人看得懂嗎
- 有沒有出現太多只有圈內人才懂的名詞
- 有沒有把結果畫面講出來
- 新手看完會不會覺得門檻下降
- 能不能直接拿去做貼文、簡報、說明頁
