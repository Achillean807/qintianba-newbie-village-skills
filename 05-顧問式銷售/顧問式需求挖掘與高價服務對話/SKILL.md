# 顧問式需求挖掘與高價服務對話

真正難的不是報價。
是在報價前，把對方到底想解什麼問題問清楚。

## 使用情境

- 客戶私訊來問價格
- 客戶說想了解一下
- 客戶問題很模糊
- 服務案單價高，不適合直接丟價目表
- 需要先釐清行銷目的、需求邊界、成功標準

## 核心觀念

### 1. 先挖目的，不先報價
如果連目標都不知道，報價只會變成比價。

### 2. 先問現況，再問想要
很多人會說自己要的是 A。
但追下去會發現他真正卡的是 B。

### 3. 問句要讓對方願意講，不是像在審問
問法要自然、可接、能把話題往前推。

### 4. 你不是要贏口才比賽
你是要把需求、風險、決策點、預算感知一起摸出來。

## 對話順序

```
1. 先接住：確認有收到，也表現出理解
2. 問目的：這次最想解的是什麼
3. 問場景：現在是什麼情況，卡在哪
4. 問結果：希望做到什麼程度才算成功
5. 問限制：預算、時間、人力、內部決策者
6. 再決定：給建議、給方向、約通話、或正式提案
```

## 高品質提問範例

### 問目標
- 這次您最想優先解的是曝光、詢問量，還是實際成交
- 如果只能先改善一件事，您現在最希望是哪一塊

### 問現況
- 您目前已經做到哪一步了
- 之前有自己試過哪些方式，哪裡最卡

### 問結果
- 您心中理想的成果畫面會長什麼樣子
- 這案子如果算成功，三個月後您最想看到什麼變化

### 問限制
- 這件事目前最大的限制是時間、預算，還是內部人手
- 除了您之外，還會有誰一起決定這個案子

## 回覆風格

- 像顧問，不像客服機器人
- 像在幫他釐清，不像在逼供
- 句子可以短，但要準
- 不要急著證明自己多厲害

## 什麼時候可以開始提案

至少要知道這四件事：
- 他要解什麼問題
- 為什麼現在要解
- 他預期的成功長什麼樣
- 誰決定、什麼時程

如果這四件還不知道，先不要丟完整提案。

## 輸出格式

### 格式 A：第一輪私訊回覆
- 接住對方
- 丟 2-3 題高品質問題
- 留下自然的下一步

### 格式 B：報價前需求盤點清單
- 目標
- 現況
- 對象
- 限制
- 時程

### 格式 C：通話前問題腳本
- 開場
- 中段追問
- 收尾確認

### 格式 D：提案前摘要
- 我理解到的需求
- 目前主要卡點
- 建議先解的順序
- 下一步安排

## 禁忌

- 不要一上來就報最便宜方案
- 不要用一堆空泛商業詞嚇人
- 不要還沒搞懂需求就急著成交
- 不要把客戶的模糊問題照單全收
- 不要讓對話只剩「好喔」「可以喔」「歡迎參考」這種廢話
